*Project15: implement sm2 2P sign with real network communication
202100460134

如果不能（估计多半没有编译环境）编译C#源代码(codes.rar)，请直接使用提供的EXE文件(SM2_2P_Server/Client.rar)，该文件就是代码的生成可执行文件。
理论上在Windows X64环境下可以独立运行（Linux也许可以，作者没有测试但被声明可以运行），直接打开EXE即可。

操作流程：
先打开服务端再打开客户端，SIGN的整个过程将会自动进行，椭圆曲线的参数无法方便地格式化为可视字符串，输出出来全是方框问号，没有输出的意义，就只给了结果。
只要不搞破坏，签名显然是成功的。
（本来可以调用库格式化，但是这样程序超宽了，压缩了也会超过25MB的网页GITHUB上传单文件大小限制，就删去了格式化库函数的调用。）
